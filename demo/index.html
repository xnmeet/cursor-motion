Velocity-Verlet Physics

Real spring simulation at 240Hz internal rate. Tip position, angle, and body dynamics all independently sprung for organic motion.

7-Candidate Path System

Direct, turn, brake, and orbit paths compete via scoring. The winner always takes the most natural route to any point.

Zero-Allocation Hot Path

Pre-allocated state objects, flat scalar fields, inlined loops. No GC pauses during animation — ever.

📦

Modular TypeScript

8 focused modules with full type declarations. Import only what you need — tree-shaking friendly, zero dependencies.

Three lines to magic.

Drop-in integration. The engine handles physics, you handle rendering — or use our DOM renderer out of the box.

View full documentation 
import { CursorMotionEngine, DomCursorRenderer }
  from 'cursor-motion-js';

// Create engine + renderer
const renderer = new DomCursorRenderer();
const engine = new CursorMotionEngine({
  onUpdate: renderer.onUpdate
});

// Move to any point — spring physics handles the rest
await engine.moveTo({ x: 600, y: 320 });
await engine.click();

From point A to delight.

Every cursor move passes through a carefully orchestrated pipeline.

1

Path Selection

7 candidate cubic Bezier paths are generated based on heading, distance, and turn demand. The lowest-score path wins.

2

Spring Progress

A Velocity-Verlet spring drives progress along the path. Response 1.4, damping 0.9 — tuned to feel intentional, never robotic.

3

Visual Dynamics

Tip position, rotation angle, body compression, and fog glow all spring independently at 240Hz for physically coherent motion.
