Why

Built for realism.

Every detail designed to produce motion that feels genuinely human.

Spring Physics

Velocity-Verlet integration at 240Hz. Underdamped springs with natural overshoot and settle.

7-Path Intelligence

Generates candidates — direct, turn, brake, orbit — and scores each for smoothness.

Zero Allocation

Flat scalars, pre-allocated outputs. No GC pressure on hot path — locked 60fps.

Visual Dynamics

Independent tip + angle springs create realistic tilt and momentum lag.

Dirty-Check DOM

Only writes transforms when values change. Zero forced reflows.

TypeScript Native

Strict types with full exports. Every config and state is properly typed.

API

Simple interface.
Full control.

Three lines to add natural cursor motion. Configure spring tension and path shape when you need precision.

  • Automatic path selection from 7 candidates
  • Configurable spring response & damping
  • Built-in DOM renderer with fog effect
  • Chainable async moves
  • Full TypeScript intellisense
TypeScript import { CursorMotion } from 'cursor-motion-js' // Create instance const cursor = new CursorMotion({ response: 1.4, damping: 0.9, }) // Move — path chosen automatically await cursor.moveTo({ x: 420, y: 280 }) // Chain with custom path params await cursor.moveTo( { x: 800, y: 150 }, { arcSize: 0.12, flow: 0.7 } )
Architecture

How it works

Three stages, every frame.

1

Generate

7 bezier candidates from direction, heading and distance context.

2

Score

Measure length, curvature energy, and turn demand. Lowest score wins.

3

Animate

Velocity-Verlet spring drives progress with tip + angle dynamics.
